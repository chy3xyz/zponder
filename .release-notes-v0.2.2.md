## Fixes

### RPC `-32005` Limit Exceeded — Improved

BSC public RPC nodes enforce strict limits on `eth_getLogs` queries. v0.2.1 introduced automatic `batch_size` halving, but the indexer could still appear to "stop" because:

1. `rpcCallWithRetry` performed 3 exponential-backoff retries on `RpcLimitExceeded`, wasting time
2. Each retry incremented the circuit breaker failure counter, quickly triggering `CircuitOpen` (30s cooldown)
3. After halving to the minimum batch_size of 100, there was no cooling period before retry

**Fix:**
- `eth_rpc.zig`: Return `error.RpcLimitExceeded` immediately without retries — the problem is batch size, not transient RPC failure
- `indexer.zig`: When `batch_size` reaches the minimum (100), sleep 10s before retrying to let the RPC node cool down
- `config-busd.toml`: Added comments recommending professional BSC RPC endpoints (Ankr, QuickNode, GetBlock)

The indexer now adapts smoothly to RPC limits without false circuit-breaker trips.
